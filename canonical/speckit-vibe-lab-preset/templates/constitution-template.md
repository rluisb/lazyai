# {{project_name}} Constitution

## WHAT

This project uses Spec Kit with the vibe-lab preset.

## HOW

- Plan with specs and tasks.
- risk-calibrated flexibility: use enough process for the risk, not ceremony by default.
- Keep the simplest viable path.
- Write the test before the change.
- Verify behavior, not plumbing.

## DON'T WANT

- Speculation about unverified integrations.
- Ceremony without observable value.
- Runtime claims beyond what is actually tested.
- Scope creep: if it wasn't asked, it isn't included.

## VALIDATE

- Every spec, plan, and task must have an acceptance check.
- Ambiguity is marked with `[NEEDS CLARIFICATION: ...]`.
- No live integration claim without observed verification.
