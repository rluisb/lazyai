# All Remaining RPI Cycles — Overview

## Cycle 2.1 — Finish semantic validation integration

Goal:

```text
Link docs, curate templates, and capture warning inventory after Cycle 2.
```

Why:

```text
Cycle 2 added semantic validation, docs, and templates. Cycle 2.1 makes them discoverable and creates a cleanup path for shipped assets.
```

## Cycle 3 — Hook lifecycle catalog

Goal:

```text
Define a neutral hook lifecycle and map adapter capability honestly.
```

Why:

```text
LazyAI has hook assets and multiple adapter surfaces, but needs a clear lifecycle/capability model without pretending every tool supports hooks equally.
```

## Cycle 4 — Trace/eval templates and rubrics

Goal:

```text
Add trace taxonomy, improvement-loop templates, and rubric assets without adding a judge/scoring/runtime engine.
```

Why:

```text
vibe-lab quality improves from trace evidence and human-gated rubric review, not hidden autonomous evals.
```

## Cycle 5 — MCP, adapter docs, compliance, token-rent

Goal:

```text
Improve MCP tool guidance, adapter capability docs, official compliance matrix, and token-rent/minimality visibility.
```

Why:

```text
Tool selection and adapter honesty are central to LazyAI’s cross-tool harness value.
```

## Cycle 6 — Context/handoff, multi-agent boundary, init/server clarity

Goal:

```text
Complete remaining guidance and clarity around context, handoff, multi-agent usage, init populate, and server capability levels.
```

Why:

```text
These topics are easy to overgrow into runtime/orchestration. The cycle keeps them as guidance, templates, and honest CLI messaging.
```
