# Context: ADRs

**Category:** Documentation
**Status:** Active

---

## Rule

Maintain accurate and up-to-date ADRs.

## Rationale

Ensures knowledge is captured and shared across the team.

## Guidelines

- **Document Decisions:** Capture why, not just what.
- **Write for the Next Engineer:** Assume no prior context.
- **Keep Docs Close to Code:** Documentation lives near what it describes.
- **Date Everything:** Decisions have dates; docs have timestamps.
- **Cross-reference:** Link related decisions and docs.

## Enforcement

- PR review checklist
- CI gates for tests and linting
