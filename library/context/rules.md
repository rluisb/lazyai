# Context: Rules

**Category:** Process
**Status:** Active

---

## Rule

Follow all established rules and guidelines.

## Rationale

Ensures consistency, quality, and traceability across the codebase.

## Guidelines

- **Read Context First:** Understand existing conventions before writing code.
- **Test as You Go:** Write tests for every non-trivial function.
- **One Task at a Time:** Complete and verify before moving to the next.
- **Follow the Plan:** Don't expand scope; create a new task if you find more work.
- **Commit Atomically:** One logical change per commit.

## Enforcement

- PR review checklist
- CI gates for tests and linting
