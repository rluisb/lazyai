# Rule: Code Review

**Category:** Process
**Status:** Active

---

## Rule

All code changes require a peer review before merging.

## Rationale

Improves code quality, shares knowledge, and catches bugs early.

## Guidelines

- **Review for Correctness:** Does the code do what it's supposed to do?
- **Review for Maintainability:** Is the code easy to read and understand?
- **Review for Security:** Are there any security vulnerabilities?
- **Review for Testing:** Are there adequate tests?
- **Review for Performance:** Are there any performance bottlenecks?

## Enforcement

- PR review checklist
- CI gates for tests and linting
